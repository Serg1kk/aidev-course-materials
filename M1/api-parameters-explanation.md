# API параметры и модели — справка

> Про прямые API-вызовы моделей: что за параметры, что крутить и зачем, плюс сравнение основных моделей 2026 года. Примеры названий параметров взяты из n8n OpenAI-ноды (они же у большинства провайдеров).

---

## Параметры API-вызова

Все эти параметры передаются в тело POST-запроса к провайдеру. Большинство — опциональные, работают дефолты. Крутить начинают, когда результат не устраивает «из коробки».

| Параметр | Что делает | Диапазон | Дефолт | Когда менять |
|----------|------------|----------|--------|--------------|
| **Temperature** (Output Randomness) | Управляет «креативностью» / случайностью ответа. 0 — детерминированно, одно и то же на одинаковый вход. Высокие значения — разнообразнее, но менее предсказуемо | 0.0 – 2.0 | 0.7–1.0 | `0` для классификации, извлечения данных, кода. `0.7–1.0` для текстов, идей. `1.2+` для брейншторма, но уже бывают галлюцинации |
| **Top P** (Nucleus Sampling) | Альтернатива температуре. Модель рассматривает только токены, суммарная вероятность которых ≤ P. `0.1` = только самые «уверенные» варианты, `1.0` = все | 0.0 – 1.0 | 1.0 | **Обычно меняют либо temperature, либо top_p**, не оба сразу. Если нужна «уверенная» модель — оставить temp=0, top_p=1 |
| **Max Tokens** (Maximum Number of Tokens) | Жёсткий лимит длины ответа в токенах. Модель остановится принудительно, даже посреди предложения | 1 – context window | зависит от модели | Ставить всегда, когда нужна предсказуемая цена или короткий ответ. 1 токен ≈ 0.75 английского слова или 0.5 русского |
| **Frequency Penalty** | Штраф за повторение токенов. Чем больше значение, тем реже модель повторяет уже использованные слова/фразы | -2.0 – 2.0 | 0 | `0.3–0.7` если модель зацикливается или повторяет одни и те же формулировки. Отрицательные значения — если наоборот нужны повторы |
| **Presence Penalty** | Штраф за упоминание темы, которую уже упомянули. Разница с frequency: presence реагирует на **факт появления**, frequency — на **частоту** | -2.0 – 2.0 | 0 | `0.3–0.6` чтобы модель расширяла темы, не зацикливалась на одном предмете |
| **N / Number of Completions** | Сколько разных вариантов ответа сгенерировать за один запрос | 1 – 128 | 1 | `3–5` для A/B-подбора (заголовков, вариантов текста). Платишь за каждый вариант отдельно |
| **Stop Sequences** | Массив строк, при появлении которых модель останавливается | — | нет | Для строгих форматов: `["\n\n", "###", "User:"]`. Частый приём в чат-обёртках |
| **Seed** | Фиксирует «случайность» — при одном и том же seed + prompt модель даёт тот же ответ (почти) | целое число | нет | Для воспроизводимых тестов, оценки промптов. Не гарантирует 100% идентичность |
| **Max Tool Calls Iterations** (agentic) | Сколько раз подряд агент может вызвать tools в рамках одного запроса, прежде чем вернуть финальный ответ | 1 – ∞ | 10–15 | Повышать если агент решает сложную задачу с цепочкой вызовов. Понижать — чтобы не улетел в долгий loop |
| **Response Format** | Формат вывода. `text` — обычный текст, `json_object` — строгий JSON, `json_schema` — JSON по заданной схеме | text / json_object / json_schema | text | Всегда `json_schema` когда нужен структурированный результат для пайплайна |
| **Tools / Function Calling** | Список функций, которые модель может вызвать вместо текстового ответа | JSON-список | нет | Для агентов, RAG, интеграций. Модель сама решает, когда вызвать и с какими параметрами |

### Полезные правила

- **Temperature + Top P**: крутить одно из двух, не оба. Провайдеры сами это рекомендуют.
- **Max Tokens = стоимость**: жёсткий лимит — жёсткий потолок по деньгам.
- **Frequency + Presence**: тонкая настройка «разнообразия». Для обычных задач не трогать.
- **Seed**: только OpenAI и некоторые OpenRouter-модели. У Claude и Gemini не всегда поддерживается.

## Основные модели (апрель 2026)

|--------|----------|------------------------|-------|-------------|
| **GPT-4.1** | 1M | ~$3 / ~$12 | Флагман общего назначения, огромный контекст | Сложные задачи, длинные документы, агенты |
| **GPT-4o** | 128K | ~$2.5 / ~$10 | Мультимодальный (текст, аудио, картинки), быстрый | Чат-интерфейсы, голос, vision |
| **GPT-4o-mini** | 128K | ~$0.15 / ~$0.6 | Дешёвый, быстрый, 80% качества 4o | Массовые задачи, классификация, обработка данных |

### Anthropic (Claude)

| Модель | Контекст | Input / Output ($/1M) | Фишка | Когда брать |
|--------|----------|------------------------|-------|-------------|
| **Claude Opus 4.6** | 1M (beta) | ~$15 / ~$75 | Флагман, лучший в кодинге и агентах | Сложный код, ресёрч, критичные задачи |
| **Claude Sonnet 4.6** | 200K / 1M | ~$3 / ~$15 | Баланс цены и качества, workhorse | Продакшн-задачи, чат-ассистенты, автоматизация |

Docs: [docs.claude.com/en/docs/about-claude/models](https://docs.claude.com/en/docs/about-claude/models)

### OpenRouter — агрегатор

[openrouter.ai](https://openrouter.ai/) — один API ключ, один endpoint, доступ к **всем** моделям выше. Платишь по цене провайдера + ~5% сверху. Удобно для демо, A/B-тестов, fallback-логики (если одна модель упала — переключиться на другую).

Формат ID: `openai/gpt-4o-mini`, `anthropic/claude-sonnet-4`, `google/gemini-2.5-flash`, `meta-llama/llama-3.3-70b`.

## TL;DR в 30 секунд

> «У API есть 3 группы параметров. **Первая — длина**: max_tokens ставит потолок ответа. **Вторая — креативность**: temperature (0 — строго, 1 — свободно). **Третья — формат**: response_format заставит модель вернуть JSON. Остальное — тонкая настройка, по дефолту не нужно.
> Моделей — три семьи: OpenAI, Anthropic, Google. Плюс open-source через Ollama. OpenRouter — единый API ко всем. Дешёвая массовка → GPT-4o-mini или Gemini Flash. Серьёзный прод → Sonnet или GPT-4.1. Reasoning → o3. Длинные документы → Gemini Pro.»
