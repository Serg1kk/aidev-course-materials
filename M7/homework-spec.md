## Секция 1 — 🔒 Безопасность + приватность

**Задача.** Представь, что ИБ проверяет твой проект. Составь для `proshop_mern` таблицу «класс данных → инструмент» и 3 строки «Риск → Решение».

**Критерий «сделано»:** ≥3 строки, в каждой — класс данных (Public/Internal/Confidential/Critical) + конкретный риск + архитектурное (не «политика») решение.

**Мини-пример (1 строка):**
| Класс | Данные в proshop_mern | Риск | Решение |
|---|---|---|---|
| Critical | данные карт / PII покупателей | передача PII в облачную модель в промпте | PII-маскирование на прокси перед отправкой ИЛИ локальная модель |

**Подсказки:**
- Пройди по своим сервисам M3-M5: что из данных Public, что Critical?
- Главное правило: free/Plus ChatGPT тренит на данных, API/Enterprise — нет.
- Решение = архитектура (изоляция/маска/RBAC в коде), а не «попросили не сливать».

**🔎 Поресёрчить →** [`7.1-security/data-classification-and-checklist.md`](./7.1-security/data-classification-and-checklist.md) (матрица + чеклист) · [`attacks-owasp-guardrails.md`](./7.1-security/attacks-owasp-guardrails.md) (что вообще может пойти не так).

## Секция 2 — 💰 Выбор стека (Cloud vs Local)

**Задача.** Возьми одну фичу `proshop_mern`, где используется LLM (например, RAG-поиск или агент M5). Сравни: текущее решение на платном API vs гипотетический переезд на локальную модель (Qwen / DeepSeek через Ollama). Прикинь, в какой момент выгоднее поднять свой сервер.

**Критерий «сделано»:** названы конкретная модель + движок (напр. Qwen3.6-27B на vLLM) + ориентир по объёму, при котором self-host окупается.

**Мини-пример:** «При <30-50M токенов/мес — остаёмся на API (self-host не окупится). Если фича вырастет до миллионов запросов и появится требование 152-ФЗ — переезжаем на Qwen3.6-27B (vLLM), break-even по LCOAI ≈ ...»

**Подсказки:**
- Формула: `LCOAI = (CapEx + OpEx) / 1000 инференсов`. GPU — только ~40% TCO.
- Не считай «счёт за токены vs цена стойки» напрямую — это вводит в заблуждение.
- Чаще всего ответ — гибрид: рутина локально, пики в облако.

**🔎 Поресёрчить →** [`7.2-economics/cost-optimization-playbook.md`](./7.2-economics/cost-optimization-playbook.md) (TCO, break-even) · [`pricing-and-deprecation-cheatsheet.md`](./7.2-economics/pricing-and-deprecation-cheatsheet.md) · [`7.1-security/local-models-inference-guide.md`](./7.1-security/local-models-inference-guide.md).

## Секция 3 — 📈 ROI (окупаемость)

**Задача.** Возьми **одну автоматизацию, которую ты построил в M3-M5** (по умолчанию — твой агент из M5: авто-откат фичи / мониторинг / n8n-воркфлоу). Посчитай, сколько человеко-часов в месяц она экономит, и переведи в деньги.

**Критерий «сделано»:** 1 конкретная цифра экономии (часы/мес) + перевод в ₽/$ по ставке.

**Мини-пример (из программы):** «Раньше релиз-инженер тратил **4 часа** на раскатку фичи + мониторинг ошибок. Теперь агент M5 делает это за ~0 мин. При 8 релизах/мес → 32 ч/мес × ставка = экономия N ₽/мес. Минус стоимость токенов агента → чистая выгода.»

**Подсказки:**
- Бери конкретную рутину, которую агент реально убрал, а не «вообще ускорил».
- Не забудь вычесть стоимость токенов/инфры — это честный ROI.
- Можешь взять любую свою автоматизацию из M3-M5, не обязательно M5-агент.

**🔎 Поресёрчить →** [`7.3-adoption/metrics-and-pitfalls.md`](./7.3-adoption/metrics-and-pitfalls.md) (как мерить, токеномика) · [`7.2-economics/cost-optimization-playbook.md`](./7.2-economics/cost-optimization-playbook.md).

## Секция 4 — 🚀 Финальный pitch (план внедрения)

**Задача.** Собери короткий питч (1 слайд / 1 абзац): как бы ты внедрил эту систему в реальную команду. Включи: (1) Architecture Overview — как M3 (логика) + M4 (интерфейс) + M5 (агент) работают вместе; (2) Roadmap внедрения на 4 недели (обучение → пилот → масштабирование); (3) мини AI-policy — 3-5 правил для коллег (что с AI можно, что нельзя).

**Критерий «сделано»:** есть roadmap (4 недели) + ≥3 правила AI-policy.

**Мини-пример AI-policy:**
1. Public-код и доки — любой AI-инструмент ок.
2. Данные клиентов / секреты — только через корпоративный API с ZDR или локально, никогда в free-tier.
3. Агент с доступом к prod — только read-only + human-подтверждение на необратимое.

**Подсказки:**
- Roadmap: не «все на AI с понедельника» (это саботаж) — пилот на 2-3 champions, потом шире.
- AI-policy держи короткой и понятной не-технарям.

**🔎 Поресёрчить →** [`7.3-adoption/adoption-playbook.md`](./7.3-adoption/adoption-playbook.md) (внедрение, champions) · [`building-agents-in-companies.md`](./7.3-adoption/building-agents-in-companies.md).
