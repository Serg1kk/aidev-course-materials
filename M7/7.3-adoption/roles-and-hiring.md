# Роли и хайринг в AI-эпоху

> Срез: май 2026. Команды меняются: состав (найм/увольнение), обучение, структура и процессы. Ключевой навык — «агентность сотрудника > агентность агента».

## 1. Типология «типов людей» — это ТРИ РАЗНЫЕ оси (не путать)

Частая ошибка — свести всё в одну линейку. На деле это три независимые оси; конкретный человек — точка в 3D:

| Ось | Категории | Про что |
|---|---|---|
| **Инженерная дисциплина** | Консерватор · Вайбкодер · AI-native инженер | как относится к инженерному суждению |
| **Стадия освоения** | Отрицающие → начинающие → овладевшие | где на личной кривой обучения |
| **Adoption-кривая команды** | Амбассадоры → ранние → «хвост» | место относительно команды |

Детализация оси «инженерная дисциплина»:
| Тип | Поведение | Что с ним делать |
|---|---|---|
| **Консерватор** | «сам пишу лучше», AI — только для пет-проектов; сопротивление на уровне идентичности | НЕ увольнять автоматом — балансирует энтузиастов; роль судьи/архитектора; работать с угрозой идентичности |
| **Вайбкодер** | «режим ковбоя», без инженерного суждения; vibe coding в проде → legacy за часы | дать инженерную дисциплину (feedback loop, ревью); изолированные задачи |
| **AI-native инженер** | разберёт задачу и архитектуру с агентом, объяснит, в чём проблема | целевой профиль найма; делать champion-ом; удерживать ростом дохода |

## 2. Вайб-кодер как новая профессия

Это НЕ «программист с AI-инструментами». Это человек, закрывающий несколько ролей сразу: бизнес-анализ + бэк + фронт + DevOps + QA + продакт-функции. Команда из 6-7 человек может схлопнуться в 1-2. Это **смена профессии целиком**, а не апгрейд. Опыт остаётся базой: на верхние уровни мастерства с нуля не прорваться. Появляется новый обязательный навык — **продажа своих услуг**.

## 3. Найм / увольнение / удержание

| Действие | Критерий | Нюанс |
|---|---|---|
| **Увольнять** | Чистый исполнитель (не ставит задачи, не прорабатывает требования) | — |
| **Увольнять** | Требует постоянного контроля и проверки результатов | — |
| **НЕ увольнять** | Душнил-консерватор | балансирует энтузиастов, тормозит безрассудный vibe coding |
| **НЕ увольнять** | Носитель tacit knowledge | неявные знания = главный блокер adoption, незаменимы |
| **Нанимать** | «Агентность сотрудника > агентность агента» | менеджерские навыки, опыт с AI, архитектура/паттерны |
| **Нанимать (AI-инженер)** | Глубина по «внутренностям» агента важнее общего ML-опыта | harness, гардрейлы, оптимизация токенов, траектории; формат собеса — live-coding агента за час |
| **Удерживать** | Сотрудник с кратным выхлопом → кратный рост дохода | иначе уйдёт к конкуренту |
| **Процесс** | Тестовые задания снова актуальны + требовать AI на собесах | возврат тестовых после эпохи «не давайте ТЗ» |

> Критерий «любит писать код» как повод для увольнения — НЕ используем: проблема не в любви к коду, а в неспособности масштабировать себя через агентов (это уже покрыто строкой «чистый исполнитель»).

## 4. Джуны — две оси, не конфликт

Кажущееся противоречие на деле — две разные плоскости:
- **Найм / инвестиции:** вкладываться в джунов имеет смысл — они активны, быстро перехватывают новые инструменты и экспертизу. Оставлять процесс только на «старичках» рискованно.
- **Обучение / риск:** джуны — зона риска over-reliance (берут результат AI, не понимая основ). Нужна дисциплина обучения: периоды работы без AI на старте, акцент на фундамент.

Вывод: **инвестируй в джунов, но с дисциплиной обучения.**

## 5. Доменная экспертиза решает

Контроль будущего — у людей на пересечении **глубокой доменной экспертизы** (накапливается годами) × **AI-флюэнси** (осваивается за месяцы — это навык, не требующий «лучшего университета»). Схема «1 CEO + миллион агентов» не работает: человеку нужна экспертиза, чтобы верифицировать решения модели (галлюцинации, контекст, техдолг). Часто нетехнические носители процесса «вайбкодят» бизнес-задачи лучше программистов — потому что знают, КАК правильно делать работу.
