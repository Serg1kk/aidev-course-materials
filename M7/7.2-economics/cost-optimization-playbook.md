# Cost-optimization playbook: кэш → routing → self-host

> Срез: май 2026. Порядок оптимизации: сначала дешёвое и быстрое (кэш, static routing), потом тяжёлое (self-host). Каждый шаг мерьте на своём профиле нагрузки.

## 1. Кэширование промптов — три разных движка

| Параметр | Anthropic | OpenAI | Google Gemini |
|---|---|---|---|
| Скидка cache-hit | ~90% | ~50% | ~75% |
| Стоимость записи | +25% (5 мин) / +100% (1 ч) | бесплатно | бесплатно (но storage-fee) |
| Storage-fee | нет | нет | да (почасовая плата за хранение кэша) |
| Активация | явная (точки кэша, ограниченное число) | автоматическая (zero-code) | явный cache-объект |
| Когда брать | агент-лупы, длинный system-prompt | «настроил и забыл» | batch/eval, длинные документы |

**Правила:**
- Стабильный контент (system prompt, документы) — в НАЧАЛО промпта; динамическое — в конец. Это и максимизирует cache-hit, и снижает TTFT.
- Один лишний пробел ломает кэш. Проверяйте hit-rate в проде: падение с 80% до 40% за неделю = инцидент юнит-экономики.
- **Стэк скидок:** кэш + Batch API (−50%) у некоторых провайдеров даёт до −95% на input. Но это **теоретический потолок** — batch-запросы могут попадать на «холодный» кэш, реальный hit ниже. На слайд: «−95% при прогретом кэше».

## 2. Routing / AI-gateway

**Лестница:**
1. **Static rules** (token-threshold / keyword / user-tier) → ловят **70-80% экономии без единой ML-модели**. Начинайте отсюда.
2. **Learned routing** (обученный роутер cost↔quality) → +15-25% сверху, окупается только на большом объёме. Эталонные бенчмарки: ~95% качества топ-модели при ~26% цены (−74%) на ряде задач.
3. **Каскад** (дешёвая → judge → дорогая) → максимум 3 уровня.

**Ловушка ложной эскалации:** если сигнал «дешёвая модель провалилась» врёт часто — вы платите трижды (дешёвый вызов + judge + дорогой). Формула честной экономии:
```
net_cost = cheap_cost + judge_cost + p_escalate × expensive_cost
```
Сравнивайте с `always_frontier`, не с `always_cheap`. Если false-fail rate > 30-50% → каскад превращается в перерасход. **Мерьте NET-экономию, не headline-rate.**

**AI-gateway landscape (май 2026):**
| Gateway | Лицензия / хост | Кому |
|---|---|---|
| OpenRouter | hosted SaaS, 300+ моделей | прототип, широкий каталог |
| LiteLLM | MIT, self-host | владельцы инфры, cost-governance |
| Portkey | Apache 2.0 (OSS) / hosted | enterprise/compliance |
| Cloudflare AI Gateway | SaaS edge | если уже на Cloudflare |
| Helicone | Apache 2.0, **maintenance mode** (приобретён, активной разработки нет) — для новых проектов не рекомендуется | — |

> Gateway даёт абстракцию над провайдером (важно для миграций при deprecation), fallback, бюджеты. Для observability проверяйте актуальность инструмента — рынок быстро меняется.

## 3. Self-host TCO vs API — честная математика

**GPU — это только ~40% TCO.** Остальное:
- over-provisioning под пик (P95): платите за мощности, которые простаивают;
- постоянная ops-нагрузка (≈20-30% времени senior-инженера);
- senior MLOps-инженер (дорого, регион-зависимо);
- model churn: флагманские open-weight модели обновляются каждые 6-8 недель → тест + валидация + миграция.

**LCOAI (правильный фрейм):**
```
LCOAI = (CapEx + OpEx) / 1000 инференсов
```
Прямое сравнение «месячный счёт за токены vs цена серверной стойки» — flawed accounting, ведёт к budget shock через полгода.

**Break-even (ориентиры, унифицировать через LCOAI):**
| Объём | Вывод |
|---|---|
| < 30-50M токенов/мес | тяжёлое железо **почти никогда не окупается** |
| SLM на consumer-GPU (~$2K) | окупается < 3 месяцев |
| 100B+ параметров на стойках | 2-5 лет |

**Decision-tree:**
| Сегмент | Рекомендация |
|---|---|
| Небольшая команда, бюджет ограничен | Managed API (speed-to-market, без six-figure LLMOps) |
| Mid-market, высокий объём | Hybrid router (фиксированная база на своём железе + переменный API на пики) |
| Regulated / healthcare / finance | Full self-host (data sovereignty важнее экономики — break-even = плата за compliance) |

> **Миф:** «скачал open-weight модель = бесплатный enterprise-деплой». Веса бесплатны; инфраструктура, обслуживание и talent — нет.

## 4. BYOK и подписки

- **BYOK** (Bring Your Own Key): приносите корпоративный ключ провайдера напрямую → платите за сырые токены, минуя наценку обёртки, и видите полный расход.
- **Ловушка подписок:** заявленная цена подписки IDE-ассистента может многократно вырасти на практике — агенты делают скрытые API-вызовы, лимит съедается, дальше дорого. Для 24/7-агентов API «съедает тысячи» → им подписка; для разовых задач BYOK выгоднее.
- В 2026 single-model подход редко оправдан — переключение между моделями тривиально на большинстве платформ. Выбирайте самую lean-эффективную технологию под задачу.
