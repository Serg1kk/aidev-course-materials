# Готовый промпт: чат-виджет в proshop_mern (DZ1, шаг 4)

> Скопируй блок ниже целиком в свой кодинг-агент (Claude Code / Cursor / OpenCode), запущенный в корне
> твоего форка `proshop_mern`. Сначала агент составит карту форка, потом сделает виджет.
> Фронт — вторичен: цель «работает и шлёт на роутер», красоту по `Design.md` наведёшь потом.

---

```
Ты работаешь в моём форке proshop_mern (MERN: React 16 + Redux + React Router v5 фронт, Express +
Mongoose бэк). Задача: добавить чат-виджет AI-ассистента, доступный ЗАЛОГИНЕННОМУ пользователю.

ШАГ 0 — карта форка (сделай сначала, выведи кратко):
- где хранится состояние логина (userInfo в Redux / localStorage), как достать текущего user (id, name, токен);
- структура frontend/src (screens, components, store);
- как фронт ходит на бэк (axios/fetch, прокси на бэкенд-порт).

ЗАДАЧА:
1. Компонент <ChatWidget/>: плавающая кнопка-пузырь в углу, по клику открывается панель чата
   (история сообщений + поле ввода + кнопка отправки). Рендерить ТОЛЬКО если пользователь залогинен.
2. При отправке POST'ить на эндпоинт роутера. Сделай URL конфигурируемым через переменную окружения
   фронта (например REACT_APP_ASSISTANT_URL), дефолт — заглушка http://localhost:5678/webhook/chat .
   Тело запроса: { "userId": "<id текущего юзера>", "message": "<текст>" }.
   Заголовок Authorization: Bearer <JWT текущего юзера>, если он есть (понадобится в DZ2).
3. Ответ роутера показывать в ленте как сообщение ассистента. Markdown в ответе можно рендерить.
4. Показывать индикатор «печатает...» во время запроса и аккуратно обрабатывать ошибку сети.
5. Виджет подключить в общий layout, чтобы он был на всех страницах залогиненного юзера.

ОГРАНИЧЕНИЯ:
- Не ломай существующий роутинг/стили. Минимальные изменения, в стиле окружающего кода.
- Не хардкодь URL и токены. Никаких секретов в коде.
- Не трогай бэкенд-логику заказов/оплаты.

КРИТЕРИЙ ГОТОВО:
- Залогинился → виден пузырь → написал «привет» → запрос ушёл на REACT_APP_ASSISTANT_URL с userId и
  message → ответ отрисовался в ленте. Для неавторизованного виджета нет.

После реализации: покажи, какие файлы создал/изменил, и как запустить.
```

---

**Заметка (не для агента):** если решишь проксировать через свой Express (а не звать webhook напрямую
из браузера) — это чище для DZ2 (бэкенд знает `req.user`, добавляет userId сам и не светит webhook в
браузер). Тогда добавь роут `POST /api/assistant/chat` (за `protect`), который форвардит на роутер.
