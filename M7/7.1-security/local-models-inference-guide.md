# Локальные модели, инференс-стек и квантизация (+ 152-ФЗ)

> Срез: май 2026. Для сценариев «данные не должны уходить за периметр» (152-ФЗ, медицина, банк, суверенность). Модели меняются ежемесячно — важна не модель, а пайплайн.

## 1. Зачем локалки: драйвер 152-ФЗ

ФИО / телефоны / сканы паспортов, отправленные в зарубежный API, = незаконная трансграничная передача персональных данных (нужно согласие субъекта + локализация копии в РФ). Медицина де-факто требует on-prem. Поэтому **в РФ спрос на локальные модели выше, чем в ЕС** — там облаков достаточно, у нас драйвер — приватность + независимость + санкционная страховка.

Прямой ответ на требование суверенности: локальная open-weight модель ИЛИ PII-маскирование на входе (см. `data-classification-and-checklist.md`).

## 3. Инференс-стек: что под какую нагрузку

| Движок | Когда брать | Сильная сторона | Статус 2026 |
|---|---|---|---|
| **vLLM** ⭐ | Прод по умолчанию | PagedAttention, continuous batching, FP8, hot-swap, нативный multi-LoRA, широчайшее железо | Дефолт прода (Linux Foundation) |
| **SGLang** | Prefix-heavy: RAG / multi-turn / agent-loops | RadixAttention (prefix-cache 75-95%) | Active |
| **TensorRT-LLM** | NVIDIA-only, модель заморожена, пиковый throughput | Компилируемые движки, +30-50% на high-concurrency | NVIDIA-only, compile-tax ~28 мин cold-start |
| **TGI** | (раньше — HF-нативный стек) | HF-интеграция | 🔴 **maintenance mode → мигрировать на vLLM** |
| **Ollama** | Локальный прототип / 1 машина | One-command, GGUF, авто-детект железа | 🟡 dev/прототип, **НЕ для прода** |
| **llama.cpp** | CPU / edge / Mac | От Raspberry Pi до Mac Studio | edge / CPU / Mac |

> **Канон:** граница прототип↔прод — порядки величин. Ollama архитектурно не делает continuous batching → тюнингом параметров разрыв не закрыть, помогает только смена движка. Маршрут: **Ollama локально → vLLM для staging/multi-user → TensorRT-LLM когда модель заморожена**.
>
> ⚠️ Throughput-цифры всегда подписывайте конфигом (модель + GPU + движок). Голое число «N токенов/с» бессмысленно — оно несопоставимо между разными моделями и железом.
>
> ⚠️ Ollama по умолчанию квантует в Q4 → это роняет tool-calling, агенты ведут себя непредсказуемо. Задавайте квант явно.

## 4. Квантизация: Q-floor под задачу

| Квант (GGUF) | Качество vs FP16 | Вердикт |
|---|---|---|
| Q8_0 | ~99% | near-lossless |
| Q5_K_M | ~97% | лучший trade-off |
| **Q4_K_M** | ~92% | **floor для кода / агентов** |
| Q3_K_M | ~85% | заметная просадка, только при нехватке памяти |
| Q2_K | ~70% | severe, только тесты |

**Правила:**
- Q4 почти не режет обычные модели, но **убивает reasoning** → **Q4 = floor для кода, Q6 = floor для reasoning-моделей**.
- **q4/b4 заметно роняет качество РУССКОГО языка** даже на больших моделях → для RU-задач держите Q6+.
- Формат под задачу: код → AWQ 4-bit (vLLM/Marlin); суммаризация → Q5_K_M+; reasoning/математика → Q6+; на Mac → MLX 4-bit.

## 5. Self-host или нет? (кратко — подробно в `7.2-economics/`)

Локалка не бесплатна: GPU — только ~40% TCO, остальное — простаивающие мощности под пик, обновление весов и senior MLOps. Ниже ~30-50M токенов/мес тяжёлое железо почти никогда не окупается. Для суверенности (regulated/healthcare/finance) self-host оправдан как плата за compliance, даже если экономически дороже. Большинство приходит к гибриду: рутина на своём железе, пики — в облако.
